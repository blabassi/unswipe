Unswipe is **CSS-first for layout**. There is no core `slidesPerView` option.

## With `unswipe/style.css`

Set `--unswipe-slide-min-width`:

| Goal | Token value |
| --- | --- |
| 1 full slide | `100%` |
| ~2 slides | `calc((100% - var(--unswipe-gap)) / 2)` |
| ~3 slides | `calc((100% - 2 * var(--unswipe-gap)) / 3)` |
| Peek of next (default) | `70%` |

```css
[aria-roledescription='carousel'] {
  --unswipe-slide-min-width: calc((100% - 2 * var(--unswipe-gap)) / 3);
}
```

<Playground
  id="sizes-two"
  label="Two slides"
  slideMinWidth="calc((100% - var(--unswipe-gap)) / 2)"
  slidesToScroll={2}
  plugins="navigation,pagination"
  slides={['Two A', 'Two B', 'Two C', 'Two D', 'Two E', 'Two F']}
  code={`[aria-roledescription="carousel"] {
  --unswipe-slide-min-width: calc((100% - var(--unswipe-gap)) / 2);
}`}
/>

Without the stylesheet, use your own rules (`flex: 0 0 50%`, grid, …).

## Optional JS helper

Prefer equal-width pages from JS? Use the [`slidesPerView` plugin](../../plugins/slides-per-view/).

<Aside type="tip">
Pair visible width with [`slidesToScroll`](../grouping-slides/) when you want `next()` / dots to advance a full page.
</Aside>
